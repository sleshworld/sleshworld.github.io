# Стресс-дневник

Локальный инструмент для заполнения заданий и ведения постоянных списков:

- диагностика убеждений и стратегий;
- дневник стрессовой ситуации;
- рационализация после дневника, когда действие снимает чувство, а не решает реальную проблему;
- подробный сценарий экспозиции с маршрутом и перечнем исключаемых действий;
- сворачиваемый отчёт внутри сценария с хронологией и динамикой чувства;
- постоянный список охранительных, избегающих и адаптивных действий;
- постоянный список триггеров с оценкой сложности для психики в процентах.

## Как открыть

На сервере: https://stress-diary-45-145-163-147.sslip.io/

Открой `index.html` в браузере.

## GitHub Pages

Сайт можно опубликовать через GitHub Pages из корня репозитория:

- repository: public;
- branch: `main`;
- folder: `/root`.

После публикации GitHub выдаст адрес вида `https://username.github.io/stress-diary-app/`.

Важно: записи привязаны к конкретному адресу сайта в конкретном браузере. Если перейти с VPS-адреса на GitHub Pages, старые записи не перенесутся автоматически. Перед переездом лучше сохранить важные записи в PDF или скачать резервную копию JSON.

## Вставка в Tilda

Готовый автономный блок находится в `TILDA-COPY-PASTE.txt`. Файл `tilda-embed.html` содержит тот же код и нужен для проверки в браузере.

1. Создай в Tilda отдельную страницу и добавь блок `T123` из категории `Другое`.
2. Открой `TILDA-COPY-PASTE.txt`, нажми `Ctrl+A`, затем `Ctrl+C` и вставь код в T123.
3. Сохрани блок и опубликуй страницу. В редакторе и предпросмотре Tilda код может отображаться как текст; приложение запускается на опубликованной странице.

Блок не загружает JavaScript или CSS с GitHub. Интерфейс изолирован от стилей Tilda, а записи используют отдельные ключи `mdpsy.stress-diary.*` в локальном хранилище браузера.

После изменений в `index.html`, `styles.css` или `app.js` пересобери блок:

```powershell
node .\build-tilda-embed.cjs
```

Для обновления сервера:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy.ps1
```

## Где хранятся данные

Записи хранятся в `localStorage` конкретного браузера и конкретного пути к файлу.

Практически это значит:

- данные не отправляются на сервер;
- пустой дневник, пустая диагностика и пустые строки списков не сохраняются;
- в другом браузере записей не будет;
- если очистить данные сайта в браузере, записи пропадут;
- кнопка `Скачать резервную копию` сохраняет записи, действия и триггеры в JSON.

## Экспорт для психолога

Внутри записи доступны:

- `Скопировать` - текст в буфер обмена;
- `Markdown` - файл с текущей записью;
- `Сохранить PDF` - открывает печать браузера; в списке принтеров выбери "Сохранить как PDF".

PDF-экспорт оформлен как отчет и скрывает пустые поля.
Постоянные списки действий и триггеров сохраняются как таблицы в альбомной ориентации; их Markdown-экспорт также имеет табличный формат.
